Hospedagem de Frontend com GitHub Pages

GitHub Pages é uma solução gratuita do GitHub para hospedar sites estáticos. Ideal para projetos frontend simples, ele permite publicar o site diretamente a partir do repositório GitHub.

Passos para Publicação com GitHub Pages

  1. No seu repositório, vá para Settings > Pages.
  2. Em "Source", selecione a branch (geralmente main) e a pasta (/root ou /docs).
  3. Salve as alterações, e seu site estará disponível em um URL gerado pelo GitHub.

Agora seu frontend está acessível e hospedado de forma gratuita. Cada vez que você fizer um commit na branch escolhida, a página será automaticamente atualizada.

Gerenciamento de Processos Node.js com PM2

PM2 é um gerenciador de processos popular para Node.js, permitindo executar, monitorar e reiniciar automaticamente aplicações.

Instalação e Uso Básico

# Instale o PM2 globalmente
npm install -g pm2

# Inicie uma aplicação
pm2 start app.js --name "minhaApp"

# Monitore a aplicação em tempo real
pm2 monit

# Reinicie a aplicação automaticamente em caso de falhas
pm2 restart minhaApp

# Salve a lista de processos para iniciar automaticamente no reinício do sistema
pm2 save

Monitoramento e Logs

PM2 oferece monitoramento em tempo real e gerencia logs, o que é útil para acompanhar o desempenho da aplicação. Com pm2 monit, você pode visualizar o uso de CPU e memória, e pm2 logs exibe logs da aplicação.

Containerização com Docker

Docker permite empacotar uma aplicação e suas dependências em um contêiner, tornando-a portátil e fácil de distribuir. Essa abordagem ajuda a garantir que a aplicação funcione da mesma forma em qualquer ambiente.

Criando uma Imagem Docker

# Exemplo de Dockerfile para uma aplicação Node.js
FROM node:14

# Configuração de diretório de trabalho
WORKDIR /app

# Copia o package.json e instala dependências
COPY package*.json ./
RUN npm install

# Copia o restante dos arquivos e inicia a aplicação
COPY . .
EXPOSE 3000
CMD ["node", "app.js"]

Salve o arquivo acima como Dockerfile na raiz do seu projeto.

Construindo e Executando o Contêiner

# Crie a imagem Docker
docker build -t minhaApp .

# Execute o contêiner
docker run -p 3000:3000 minhaApp

Agora a aplicação está rodando em um contêiner e pode ser acessada localmente pela porta 3000.

Orquestração com Docker Compose

Docker Compose permite gerenciar múltiplos contêineres em um só arquivo de configuração, ideal para aplicações que dependem de outros serviços, como um banco de dados.

Exemplo de Arquivo Docker Compose

# docker-compose.yml
version: '3'

services:
  app:
    build: .
    ports:
      - "3000:3000"
  db:
    image: mongo
    ports:
      - "27017:27017"

Iniciando os Contêineres com Docker Compose

# Inicie todos os contêineres definidos no docker-compose.yml
docker-compose up

# Parar os contêineres
docker-compose down

O arquivo docker-compose.yml acima configura um contêiner para a aplicação Node.js e outro para um banco de dados MongoDB, tornando o ambiente completo fácil de iniciar e gerenciar.
