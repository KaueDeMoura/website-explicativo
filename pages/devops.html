DevOps - GitHub Pages e Docker

O papel do Node.js no Backend

O Node.js é uma plataforma construída sobre o motor de execução JavaScript do Chrome, o V8. Ele permite criar servidores eficientes e escaláveis, graças ao seu modelo de execução assíncrono e orientado a eventos. Isso significa que o Node.js é excelente para aplicações em tempo real e altamente escaláveis.

Utilizando o GitHub Pages para Hospedagem

Utilizando o GitHub Pages para Hospedagem

O GitHub Pages é o serviço de hospedagem de sites estáticos do Github. Ele usa arquivos HTML, CSS e JS diretamente de um repositório, executa os arquivos por meio de um processo e publica um site.

  1. Crie um repositório no GitHub com o nome username.github.io.
  2. Clone o repositório para sua máquina local.
  3. Adicione os arquivos do seu site (HTML, CSS, JS) ao repositório.
  4. Realize um commit e envie (push) para o GitHub.
  5. O seu site estará disponível em https://username.github.io.

Para mais detalhes, você pode consultar a documentação oficial do GitHub Pages aqui.

Criando uma Imagem Docker para o Aplicativo

Criando uma Imagem Docker para o Aplicativo

A containerização com o Docker permite criar ambientes isolados, chamados contêineres, para rodar sua aplicação de forma consistente, independente do ambiente. Isso garante que seu aplicativo funcione da mesma forma em qualquer lugar.

Para criar uma imagem Docker do seu aplicativo, você deve criar um arquivo chamado Dockerfile. Este é um exemplo básico de um Dockerfile para um aplicativo Node.js:

Exemplo de Dockerfile

            
              # Imagem base do Node.js
              FROM node:22.8
              
              # Define o diretório de trabalho
              WORKDIR /app
              
              # Copia o código da aplicação
              COPY . .
              
              # Instala as Deps
              RUN npm i
              
              # Expõe a porta
              EXPOSE 3000
              
              # Comando para iniciar a aplicação
              CMD ["npm", "start"]

Esse Dockerfile cria uma imagem com o Node.js, copia seus arquivos para o contêiner, instala as dependências e inicia o servidor.

Para construir a imagem Docker, basta executar o seguinte comando no terminal: 

            
            docker build . -t (Nome-do-App)

Utilizando Docker Compose para Orquestrar Múltiplos Contêineres

Utilizando Docker Compose para Orquestrar Múltiplos Contêineres

O Docker Compose é uma ferramenta para definir e executar aplicativos Docker compostos por múltiplos contêineres. Ele utiliza um arquivo chamado docker-compose.yml para definir todos os contêineres e serviços necessários para a aplicação.

Aqui está um exemplo de como utilizar o Docker Compose para iniciar um contêiner para a aplicação Node.js e outro para o banco de dados PostgreSQL:

Exemplo de docker-compose.yml

                  
                  version: '3'
      
                  services:
                    app:
                      build: .
                      ports:
                        - "3000:3000"
                      depends_on:
                        - postgres
                      environment:
                        - DB_HOST=postgres
                        - DB_PORT=5432
                        - DB_USER=(seuUSer)
                        - DB_PASSWORD=(suaSenha)
                        - DB_NAME=(nomeDoBanco)
      
                    postgres:
                      image: postgres:latest
                      environment:
                        - POSTGRES_USER=(seuUSer)
                        - POSTGRES_PASSWORD=(suaSenha)
                        - POSTGRES_DB=(nomeDoBanco)
                      volumes:
                        - ./data:/var/lib/postgresql/data
                      ports:
                        - "5432:5432"

Neste exemplo, o serviço app é construído a partir do Dockerfile, e o serviço postgres usa uma imagem pronta do PostgreSQL. O serviço app depende do PostgreSQL e o Docker Compose irá garantir que o PostgreSQL esteja em funcionamento antes de iniciar a aplicação.

Para iniciar os contêineres com o Docker Compose, basta executar o seguinte comando: 

                    
                    docker-compose up

Isso irá iniciar os dois contêineres e sua aplicação será acessível em http://localhost:3000.
